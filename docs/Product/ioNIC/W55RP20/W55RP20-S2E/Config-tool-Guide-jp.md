## 概要  

このドキュメントは、Configツールの詳細な使用方法を提供します。  

## Configツールのサポートリストとダウンロード  

Wiznet Configurationツールは、ネットワーク上の製品を**検索、設定変更、ファームウェアアップロード**などを可能にする設定ツールです。  

* [**サポートデバイスリスト**](https://github.com/Wiznet/WIZnet-S2E-Tool-GUI#support-devices)  

Configurationツールは、Github上で「WIZnet-S2E-Tool-GUI」という名前で公開されています。このツールはWIZnetのGithubページで利用可能で、リポジトリのリリースページから最新バージョンの実行ファイルをダウンロードできます。  

Pythonをベースに構築されており、任意のOSで動作します。  

- **Githubリポジトリ: [WIZnet-S2E-Tool-GUI Githubリポジトリ](https://github.com/Wiznet/WIZnet-S2E-Tool-GUI)**  
- **最新バージョンのダウンロード: [WIZnet-S2E-Tool-GUI Githubリポジトリ: リリース](https://github.com/Wiznet/WIZnet-S2E-Tool-GUI/releases)**

### ① 上部アイコンメニュー  

- Configurationツールの上部に位置し、製品の検索、ファームウェアのアップロード、再起動、初期化などが可能です。  

|                                                                              |  
| ---------------------------------------------------------------------------- |  
| ![](/img/products/s2e_module/wiz5xxsr-rp/configuration_tool_manual/menu_icons.png) |  
| 図: **アイコンメニュー**                                                    |  

- #### 1) デバイス検索  

- 同じネットワーク上の接続デバイスを検索します。  
- UDPブロードキャスト（ネットワーク内のモジュールを検索、複数のモジュール製品を検索可能）およびTCPユニキャスト（単一のモジュールを検索）で検索できます。  
- **ネットワークを使用したデバイス検索ではTCP/UDPポート50001を使用します**。ファイアウォールやウイルス対策ソフトがそのポートをブロックしている場合、デバイス検索が失敗する可能性があります。  
- このような場合、ファイアウォールやウイルス対策ソフトを無効にしてからテストすることをお勧めします。  
- 複数のネットワークアダプタを使用している場合、ネットワークインターフェイスのメトリック優先度によりパケット転送が失敗することがあります。<br />  
OS設定で使用しているネットワークアダプタ以外をすべて無効にし、再度テストしてください。<br />  
VMwareやVirtual Boxなどの仮想マシン（VM）を使用している場合、VMのネットワーク用に作成された「仮想イーサネットアダプタ」によってこの状況が引き起こされる可能性があります。<br />  
VMwareやVirtual Boxなどの仮想マシンを無効にするか、TCPユニキャストで検索してください。  

- #### 2) 設定適用  

- 変更した設定を保存し、デバイスに適用します。  
- 設定変更が成功すると、Configurationツールに反映されます。  
- 操作完了後、製品は自動的に再起動します。  

- #### 3) ファームウェアアップロード  

**ファームウェアのアップロード中に電源を切らないでください**  
**これにより、製品が正常に動作しなくなる可能性があります**  

- Wiznetが提供するファームウェアバイナリファイルを選択し、製品のファームウェアを更新します。  
- 操作完了後、製品は自動的に再起動します。  
- ファームウェアのアップロードが成功すると、以下のメッセージが表示されます。  

|                                                                                           |  
| ----------------------------------------------------------------------------------------- |  
| ![](/img/products/wiz750sr/guiconfigtoolmanual/gui_configtool_firmwareupload_success.png) |  
| 図: **ポップアップメッセージ - ファームウェアのアップロード成功**                   |  

- **ネットワークを介したデバイスファームウェアのアップロードにはTCPポート50002を使用します。** ファイアウォールやウイルス対策ソフトがそのポートをブロックしている場合、デバイスファームウェアのアップロードが失敗する可能性があります。<br />  
このような場合、ファイアウォールやウイルス対策ソフトを無効にしてからテストすることをお勧めします。  
- ファームウェアを更新する際、Configツールを実行しているPCのIP帯域がデバイスと一致している必要があります。<br />  
DHCPモード（自動IP割り当て）を使用している場合、PCと製品が同じルーターからIP割り当てを受け取るように設定してください。<br />  
静的モード（直接IP割り当て）を使用している場合、以下の例のように設定してください:<br />  
例) デバイスIP: 192.168.**11**.2<br />  
PC IP: 192.168.**11**.3 (同じクラスCプライベートIP帯域内の別のIPアドレス)<br />  
- 誤ったファームウェアでアップロードを行った場合、製品の正常な動作は保証できません。  

- #### 4) デバイスのリセット  
- 選択したデバイスを再起動します。  

- #### 5) 工場出荷時設定にリセット  
- デバイスの設定を工場出荷時のデフォルト値にリセットします。  
- 操作完了後、デバイスは自動的に再起動します。  

- #### 6) 設定の保存  
- 選択したデバイスのすべての現在の設定をファイルに保存します。  
- 環境の変更による再設定が必要な場合や、現在の設定を別のデバイスに適用する際に便利です。  

- #### 7) 設定の読み込み  
- 「設定の保存」機能で保存したファイルから設定を読み込みます。  
- 読み込み後、「設定適用」ボタンを押してこれらの設定をデバイスに適用してください。  

- #### 8) 終了  
- Configurationツールを終了します。

### ⑥ 一般オプション

- 製品の詳細設定の表示と変更
- 基本設定 / オプション / MQTTオプション / 証明書マネージャータブで設定を構成します (W55RP20-S2Eの場合)
|                                                                                      |
| ------------------------------------------------------------------------------------ |
| ![](/img/products/s2e_module/wiz5xxsr-rp/configuration_tool_manual/basic_settings_tab.png) |
| 図: **基本設定タブ**                                                       |

#### 1) デバイス情報

- Typeは選択されたデバイスの名前を表示し、Versionはデバイスの**ファームウェアバージョン**を表示します。

#### 2) 検索識別コード

- **範囲と区切り文字: 最大8バイトの文字列**
- デフォルト: Null (使用しない)
- 検索識別コードは、製品検索時の識別用オプションです。
- このオプションが設定されている場合、製品を検索する際に設定した同じコードを入力する必要があります。そうしないと、検出されたデバイスリストに表示されません。

#### 3) ネットワーク構成

- **静的 (Static)** (デフォルト)
- 静的IPアドレス割り当てモードを選択します。このオプションを選択すると、ローカルIP、サブネットマスク、ゲートウェイ、およびDNSサーバーフィールドが入力可能になります。<br />パブリックIPアドレスとプライベートIPアドレスの両方を設定できます。
- TCPクライアントまたはTCP混合モードを使用している場合、データを転送するリモート宛先ホストがIPアドレスではなくドメイン名の場合、DNSアドレスを設定する必要があります。
- **DHCP**
- ルーター (NATルーター) に内蔵されたDHCPサーバーを通じてIPアドレスを自動的に取得する方法です。<br />一般的には、ルーターの動的IP割り当て範囲からプライベートIPアドレスを割り当てますが、パブリックIPアドレスを割り当てることも可能です。
- DHCPサーバー側は、IPアドレス、ゲートウェイアドレス、サブネットマスク、およびDNSアドレスを割り当てるため、設定を行う必要はありません。

|                                                                                      |
| ------------------------------------------------------------------------------------ |
| ![](/img/products/s2e_module/wiz5xxsr-rp/configuration_tool_manual/options_tab.png) |
| 図: **オプションタブ**                                                              |

#### 1) TCPタイムアウト
- TCP再送信のリトライ回数を設定できます。
- 値: 1〜255

#### 2) ステータスピン
- フロー制御機能DTR、DSRを有効にします。
#### 3) シリアルデバッグ
- このオプションを設定すると、デバッグUARTが製品情報やエラー状態についての情報を出力します。
- デバッグUARTのシリアル設定は固定で、**921600-8-N-1:None** です。
- **Enable with Data** オプションを設定すると、シリアルからイーサネット（S2E）またはイーサネットからシリアル（E2S）のデータを監視できます。

#### 4) シリアルコマンドモード
- このオプションを有効にすると、シリアルデータ受信中にコマンドモード切り替えコードを受信した際、データ送信モード（GWモード）からシリアルコマンドモード（ATモード）に切り替わります。デフォルト設定：有効
- シリアルコマンドモードでは、2バイトのシリアルコマンド（AT CMD）を使用して製品のさまざまな設定を変更および監視できます。
- シリアルコマンドモードに切り替えると、既存のTCP接続は切断されます。
- トリガーコード
  - 設定したトリガーコードをシリアルデータとして入力するとATモードに入ります。
  - デフォルト：[2B][2B][2B] (+++)
  - 各バイトの値はHEXコードのみを認識します。

#### 5) 接続パスワード（TCPサーバーモードのみ）
- 最大8バイトの文字列
- デフォルト：Null（未使用）
- 接続パスワードはTCPサーバーモードでのみ有効にできます（TCP混合モードでTCPサーバーとして接続する場合も可能）。
- TCPクライアントがデバイス接続後5秒以内に接続パスワードを送信しない場合、TCP接続は自動的に切断されます。

#### 1) MQTT認証情報
- ユーザー名、パスワード
- 最大128文字

#### 2) MQTTオプション
- クライアントID
  - 最大128文字
  - 空白に設定可能だが推奨されない
- キープアライブ（Keep Alive）
  - 0から65535まで設定可能
  - 使用しない場合は0に設定
- QoS
  - 0、1、2から選択可能

#### 3) MQTTトピック
- パブリッシュトピック
  - 最大128文字
- サブスクライブトピック
  - 最大128文字
  - 最大3つのサブスクライブトピックを設定可能

#### 1) ルートCA
- オプション：None、Optional、Verifyの3つの選択肢

- 有効化チェックボックス：有効にする場合はチェック

- ファイル読み込み：事前に保存された秘密鍵ファイルを読み込む
